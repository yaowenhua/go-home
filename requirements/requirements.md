# 返乡日记（go-home）— 产品需求文档

> **版本**: v2.0  
> **日期**: 2026-07-05  
> **作者**: PM Team  
> **状态**: 初稿  
> **前身**: 返乡（Return Home）v1.0  
> **本版定位**: Vue 3 重写升级版 — 更轻盈、更美观的人生倒计时日记

---

## 一、产品概述

### 1.1 产品背景

现代人的生活节奏越来越快，每天都在追逐各种目标，却很少停下来审视："我的时间还剩多少？" 大部分人知道要"珍惜时间"，但缺乏一个直观、持续的提醒机制来感知生命的流逝。

"返乡日记"（go-home）正是一款围绕这一核心洞察而设计的人生倒计时日记 Web 应用。它是"返乡"（Return Home）的升级重写版——前端从 React 全面迁移至 Vue 3，带来更流畅的交互、更优美的动画和更轻盈的体量。

它不是一个简单的倒计时工具，而是一个引导用户记录每一天、反思每一天、珍惜每一天的**生活伴侣**。

### 1.2 产品定位

**一句话描述**: 时刻提醒你生命还剩多少天，并鼓励你记录好每一天的人生倒计时日记。

**核心价值主张**:
- **感知时间**: 将抽象的生命长度转化为具体的"剩余天数"，让时间流逝变得可见、可感
- **记录日常**: 像写待办清单一样简单，记录每天做过的事，形成个人生命日志
- **分类归档**: 用标签归类生活——工作、生活、学习、旅行……让记录更有条理
- **满意度评估**: 通过星级评分，帮助用户回顾和反思每一天的"质量"
- **离线可用**: 你的日记永远在你手中，无网络也能畅快记录

### 1.3 产品目标

| 目标 | 描述 | 衡量指标 |
|------|------|----------|
| G1: 建立时间感知 | 让用户每天至少看到一次倒计时，形成对剩余时间的主动感知 | 日活跃用户（DAU）≥ 启动用户的 60% |
| G2: 促进每日记录 | 鼓励用户记录每天的关键事件，养成回顾习惯 | 日均记录数 ≥ 1.5 条/活跃用户 |
| G3: 提升生活满意度 | 帮助用户通过星级评估反思生活质量，形成正向反馈 | 用户留存率：7日≥40%，30日≥20% |
| G4: 极简体验 | 用户从打开到完成一次记录不超过30秒 | 单次任务完成时间≤30秒 |
| G5: 离线优先 | 用户无网络也能正常记录，网络恢复后自动同步 | 离线功能可用率 100% |

## 三、用户故事

### P0 级别用户故事

#### F01 — 出生日期设置

> **US-F01**: 作为新用户，我希望首次进入时系统提示我输入出生日期，这样系统才能准确计算出我的剩余天数并呈现生动的倒计时。

**验收标准**:
- [ ] 用户首次访问时，显示全屏沉浸式引导页面（Onboarding）
- [ ] 提供日期选择器（年月日），默认值为 1990-01-01
- [ ] 年龄校验：出生日期不能早于1900年，不能晚于当天
- [ ] 提交后数据同时存储到服务端 SQLite 和前端 localStorage/IndexedDB
- [ ] 录入完成后自动跳转到主页
- [ ] 用户再次访问时不再显示引导页
- [ ] 引导页有精美动画和柔和过渡效果（Vue Transition）

#### F02 — 核心倒计时展示

> **US-F02**: 作为日活用户，我希望在首页看到醒目的剩余天数倒计时，这样每次打开应用时我都能直观感受到生命的流逝，从而更珍惜时间。

**验收标准**:
- [ ] 首页顶部展示倒计时数字，字体大小至少占屏幕宽度的60%
- [ ] 显示格式：**X,XXX 天**（千位分隔符）
- [ ] 同时显示补充信息：剩余 XX 年 XX 月 / 已用 XX 年 XX 月
- [ ] 数字每 60 秒自动刷新（分钟级精度）
- [ ] 计算逻辑：以当前日期和用户出生日期之间（预期寿命天数）的天数差减去已用天数
- [ ] 倒计时数字颜色：#FF4757（醒目红色系）或 #FF6B6B
- [ ] 如果用户已超过预期寿命，显示"超额生存"提示和已超过天数
- [ ] 页面切换回首页时倒计时自动同步更新

#### F03 — 每日记录添加

> **US-F03**: 作为活跃用户，我想要以简单方式记录当天做过的事情（每件事一条），这样我可以快速记录生活中的重要瞬间，不用花太多时间。

**验收标准**:
- [ ] 首页提供输入框，placeholder 为"今天做了什么？"
- [ ] 输入框支持多行文字（最多500字符），自动调整高度
- [ ] 点击提交按钮（或 Ctrl+Enter）提交记录
- [ ] 提交成功后输入框清空，新记录立即出现在记录列表顶部
- [ ] 记录自动标记当天的日期和时间
- [ ] 可连续添加多条记录

#### F04 — 满意度评分（1-5星）

> **US-F04**: 作为用户，我希望能为每条记录进行1-5星的满意度评分，这样我可以评估每件事的"质量"，未来回顾时能知道哪些事情让我感到充实。

**验收标准**:
- [ ] 输入框下方或记录卡片上显示5颗星的可点击选择器
- [ ] 点击星星选择评分（1-5星），鼠标悬停时显示预览效果
- [ ] 评分在提交记录时一并保存
- [ ] 记录列表中的每条记录旁显示所选评分（实心星表示已选）
- [ ] 不强制评分（默认3星），鼓励用户主动评分
- [ ] 评分在编辑时可修改

#### F05 — 记录列表展示

> **US-F05**: 作为用户，我想要按日期倒序查看我所有的记录列表，这样我可以方便地回顾最近的生活状态和重要事件。

**验收标准**:
- [ ] 记录按日期分组展示（今日、昨日、更早日期分组）
- [ ] 每组显示日期标签，当日组显示"今日"
- [ ] 单条记录展示：日期时间、内容文本、星级评分、分类标签
- [ ] 列表支持滚动加载（分页，每次加载20条）
- [ ] 每条记录卡片包含编辑和删除按钮
- [ ] 记录列表采用 Vue Transition 实现平滑的列表动画

#### F06 — 数据持久化（SQLite + 本地缓存）

> **US-F06**: 作为用户，我写下的一切都应该被妥善保存，无论是否有网络连接，这样我的生活日志在任何情况下都是完整的。

**验收标准**:
- [ ] 后端使用 SQLite 存储所有用户数据
- [ ] 前端使用 localStorage（轻量）或 IndexedDB（大数据）缓存最近数据
- [ ] Offline-First: 无网络时前端缓存写入，网络恢复后自动同步到后端
- [ ] 启动时自动创建数据库和表结构
- [ ] 所有 CRUD 操作均异步写入，确保不阻塞 UI
- [ ] 数据操作失败时有错误提示和重试机制
- [ ] 数据库文件路径可配置，默认存储在项目 data 目录下

### P1 级别用户故事

#### F07 — 日历历史浏览

> **US-F07**: 作为回忆型用户，我希望能通过日历选择器浏览某一天的所有记录（只读），这样我可以回顾特定日期的生活点滴。

**验收标准**:
- [ ] 历史页面提供日历组件(/history)，可跳转到任意历史日期
- [ ] 选中日期后，页面切换到当日记录视图（只读模式）
- [ ] 有"回到今日"快捷按钮
- [ ] 日历组件高亮显示有记录的日期（小圆点标记）
- [ ] 日历切换有平滑的月份过渡动画

#### F08 — 编辑/删除记录

> **US-F08**: 作为普通用户，我有时候会写错内容或改变对某件事的评价，因此我希望能够编辑或删除已提交的记录。

**验收标准**:
- [ ] 每条记录卡片上有编辑和删除图标按钮
- [ ] 点击编辑进入编辑模式，修改内容和星级后保存
- [ ] 分类标签也可在编辑时修改
- [ ] 点击删除弹出确认对话框（"确定删除这条记录吗？"）
- [ ] 确认删除后数据从数据库和界面彻底移除

#### F13 — 分类标签

> **US-F13**: 作为有组织的用户，我希望为每条记录添加分类标签（如工作、学习、旅行），这样我可以通过标签筛选和回顾特定类型的记录。

**验收标准**:
- [ ] 记录输入表单中提供标签选择器（单标签或多标签）
- [ ] 预设标签：工作 💼、生活 🏠、学习 📚、旅行 ✈️、健康 💪、社交 🫂、美食 🍜、其他 📌
- [ ] 支持自定义标签（设置页面中添加）
- [ ] 记录卡片上显示标签徽章，颜色区分
- [ ] 标签支持筛选：点击标签可筛选显示对应标签的记录
- [ ] 标签数据持久化存储

#### F14 — 设置页面

> **US-F14**: 作为对自己有不同规划的用户，我希望能够调整预期寿命和出生日期，这样倒计时更符合我的个人情况。

**验收标准**:
- [ ] 设置页面(/settings)可编辑个人信息
- [ ] 编辑项：出生日期、预期寿命（默认80岁，范围60-120）
- [ ] 修改后倒计时立即重新计算
- [ ] 修改需要确认（"修改后将重新计算倒计时"）
- [ ] 显示当前倒计时概览信息（已用天数、剩余天数、已用百分比）

#### F15 — Offline-First 离线缓存

> **US-F15**: 作为在地铁/山区/飞机上使用的用户，我希望在没有网络的时候也能正常记录和查看，这样我不会因为网络问题错过记录生活的任何一刻。

**验收标准**:
- [ ] 首次加载后，最近90天的记录缓存到前端
- [ ] 离线时添加/编辑/删除操作写入前端缓存，标记为"待同步"
- [ ] 网络恢复后，自动将待同步数据提交到后端
- [ ] 离线时页面正常显示，无"网络错误"提示
- [ ] 网络恢复后显示"已同步"提示（非侵入式）

### 7.2 路由配置（Vue Router）

```javascript
// src/router/index.js
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  {
    path: '/onboarding',
    name: 'Onboarding',
    component: () => import('@/views/OnboardingView.vue'),
    meta: { requiresOnboarding: false }
  },
  {
    path: '/',
    name: 'Home',
    component: () => import('@/views/HomeView.vue'),
    meta: { requiresOnboarding: true, transition: 'slide-fade' }
  },
  {
    path: '/history',
    name: 'History',
    component: () => import('@/views/HistoryView.vue'),
    meta: { requiresOnboarding: true, transition: 'slide-left' }
  },
  {
    path: '/settings',
    name: 'Settings',
    component: () => import('@/views/SettingsView.vue'),
    meta: { requiresOnboarding: true, transition: 'slide-left' }
  },
  {
    path: '/:pathMatch(.*)*',
    redirect: '/'
  }
]

const router = createRouter({
  history: createWebHistory(),
  routes
})

// 导航守卫：未引导用户强制跳转到 Onboarding
router.beforeEach((to, from, next) => {
  const hasOnboarded = localStorage.getItem('go-home_onboarded')
  if (to.meta.requiresOnboarding && !hasOnboarded) {
    next({ name: 'Onboarding' })
  } else if (to.name === 'Onboarding' && hasOnboarded) {
    next({ name: 'Home' })
  } else {
    next()
  }
})

export default router
```

| 用户数据丢失（无登录） | 高 | 中 | 首页提示数据存储方式；P2 版本加入登录同步 |
| 80岁预期寿命过于绝对 | 中 | 低 | 新增设置页面，用户可自定义预期寿命（60-120） |
| 每日记录的黏性不足 | 高 | 中 | MVP 上线后通过用户反馈决定是否增加通知提醒功能 |
